## 仓库分工

```text
ComicReader_HarmonyOS：App 工程仓库。
ComicReader_Rules：远程规则仓库。
```

App 不读取发布页附件，也不读取压缩包。App 默认只读取主分支里的：

```text
generated/index.json
```

发布页和压缩包主要给人下载检查，不能代替 `generated/index.json`。

## 规则生成流程

当前流程：

```text
运行工作流
→ 种子发现（从29+个漫画站抓取站内链接）
→ SearXNG 搜索（免费无限制，最优先）
→ DuckDuckGo HTML 兜底（连续403自动跳过）
→ Brave API 搜索（付费，宽泛查询）
→ Serper API 搜索（付费，分页翻页）
→ Google CSE 搜索（付费，备选）
→ 候选去重 + 前置过滤（屏蔽非漫画域名）
→ 审计候选（检测登录/付费早退出，省HTTP请求）
→ build_index_from_report.py 生成索引
→ sanitize_rule_outputs.py 清洗非漫画源 + 重写 .ets
→ update_manifest.py 更新版本入口
→ 合并手工稳定规则
→ 写入 generated/index.json
→ 同步 rules/index.json
→ 提交到主分支
→ 打标签 + 发布 Release
→ 上传 artifact
```

其中最关键的是"合并手工稳定规则"。自动搜索可能因为搜索接口限制、站点结构变化、公开页面不合格而生成 0 条规则。如果没有手工稳定规则兜底，App 远程更新后就会显示 0 条。

## 已处理过的问题

### 1. 工作流重复

仓库里曾经同时存在两个工作流，名字相同，容易点错。旧工作流只生成文件，不发布；新版工作流才是一键运行并发布。现在旧工作流已删除，只保留新版。

### 2. 推送被拒绝

报错内容类似：

```text
推送被拒绝，需要先拉取远程主分支
```

原因是工作流运行期间，主分支被其它提交更新了。已处理：工作流提交前会自动拉取远程主分支并重新整理提交，推送失败会重试，并且加了并发锁，避免多个规则流程互相抢推送。

### 3. 发布附件重名

报错内容类似：

```text
附件名称已存在
```

原因是同时上传 `generated/index.json` 和 `rules/index.json` 时，发布页里都会变成同名的 `index.json`。已处理：发布附件改为唯一名称：

```text
generated-index.json
rules-index.json
rulebot-report.json
GeneratedSourceRules.ets
```

### 4. 流程成功但规则数量为 0

流程成功只能说明脚本运行完成，不代表 App 有可用规则。必须检查：

```text
generated/index.json 里的 rules 数组数量
audit.totalCount
```

现在已经加入 `rules/manual/index.json`，自动规则为 0 时也会把手工稳定规则放进最终索引。

### 5. build_index_from_report.py NameError: safe_str 未定义

**时间**：2026-07-06

**现象**：workflow "生成公开规则"步骤报错 `NameError: name 'safe_str' is not defined`

**原因**：`build_index_from_report.py` 第76行调用了 `safe_str()` 函数，但该函数只在 `sanitize_rule_outputs.py` 中定义，未在 `build_index_from_report.py` 中定义。

**修复**：在 `build_index_from_report.py` 中添加 `safe_str()` 函数定义。

## 每次发布后检查

发布后先看这三个位置：

```text
generated/index.json
rules/index.json
发布页附件
```

重点看：

```text
rules 是否为空
audit.generatedCount 是多少
audit.manualCount 是多少
audit.totalCount 是多少
```

只要 `totalCount` 大于 0，App 才有远程规则可加载。

## App 侧使用方法

App 内进入：

```text
设置 → GitHub 远程规则 → 从 GitHub 更新规则
```

看到"远程规则更新成功：X 条"后，再回搜索页搜索漫画名。

如果远程规则条数为 0，App 仍会使用内置本地规则，但远程规则仓库本身就没有发挥作用。

## 不做的事情

规则仓库只处理公开网页和公开图片地址，不做这些事情：

```text
不登录
不付费
不绕验证码
不破解加密接口
不伪造专用客户端协议
不做反爬绕过
```

如果某站点必须登录、必须付费、必须客户端解密，规则仓库不会生成它的可用图片规则。
